# Glossary — Smriti

## Legal Domain Terms

| Term | Meaning |
|------|---------|
| **AIR** | All India Reporter — a major Indian law report series (e.g., "AIR 2020 SC 1234") |
| **Bench** | Panel of judges hearing a case. Types: Single Judge, Division Bench (2), Full Bench (3+), Constitution Bench (5+) |
| **Case Type** | Classification: Criminal Appeal, Civil Appeal, Writ Petition, SLP, etc. |
| **Citation** | Unique identifier for a reported case (e.g., "(2020) 5 SCC 1" or "AIR 2020 SC 1234") |
| **Coram** | Number of judges on the bench (e.g., coram_size=5 means Constitution Bench) |
| **DPDP** | Digital Personal Data Protection Act, 2023 — India's data protection law |
| **Equivalent Citation** | Same case reported in different law reports (e.g., SCC and AIR versions of same judgment) |
| **FIR** | First Information Report — initial complaint to police |
| **Headnote** | Editor's summary of a case's legal principles, placed at the top of reported judgments |
| **HITL** | Human-in-the-Loop — agent pauses for user review/approval before continuing |
| **IndianKanoon (IK)** | Popular Indian legal search engine; Smriti uses its API for external document retrieval |
| **Judgment** | The court's written decision in a case |
| **MANU** | Manupatra citation format (e.g., "MANU/SC/1234/2020") |
| **Neutral Citation** | Court-assigned citation independent of law reports (e.g., "2024:INSC:123") |
| **Obiter Dicta** | Remarks by a judge that are not binding precedent |
| **Order** | A procedural directive by a court (distinct from a judgment) |
| **Overruled** | A later judgment declares an earlier one incorrect — the earlier precedent is no longer binding |
| **PAN** | Permanent Account Number — Indian tax ID (format: AAAAA9999A) |
| **Precedent** | A past judicial decision that guides future cases |
| **Ratio Decidendi** | The core legal reasoning that makes a judgment binding precedent |
| **Reportable** | Whether a judgment is marked for publication in law reports |
| **SCC** | Supreme Court Cases — India's most authoritative law report series |
| **SLP** | Special Leave Petition — an appeal to the Supreme Court |
| **Statute** | A law enacted by legislature (e.g., Indian Penal Code, CPC) |
| **Section** | Numbered division of a statute (e.g., "Section 302 IPC") |
| **Treatment** | How a later case treats an earlier one: followed, distinguished, overruled, etc. |
| **Writ Petition** | A case filed directly under constitutional powers (Articles 32/226) |

## Technical / Architecture Terms

| Term | Meaning |
|------|---------|
| **Agent** | LangGraph-based autonomous pipeline that performs multi-step legal research |
| **Chunk** | A segment of case text (typically 2000 chars with 200 overlap) stored as a vector |
| **Circuit Breaker** | Pattern that stops calling a failing service after N failures, waits before retrying |
| **Community** | Cluster of related cases identified by graph algorithms (Louvain/label propagation) |
| **Contextual Embedding** | Embedding with surrounding section context prepended to improve retrieval |
| **CRAG** | Corrective RAG — retrieval evaluation step that checks relevance and triggers refinement |
| **Dense Section** | Case sections (ANALYSIS, RATIO, ORDER, DISSENT) chunked at 1200/300 for higher precision |
| **Element Decomposition** | Breaking a legal question into testable elements (e.g., actus reus, mens rea) |
| **Embedding** | Dense vector representation of text (1536-dim via Gemini) for semantic similarity search |
| **FTS** | Full-Text Search — PostgreSQL `tsvector`/`tsquery` keyword search |
| **Hybrid Search** | Combining semantic (vector) search with keyword (FTS) search |
| **LangGraph** | Framework for building stateful, graph-based agent workflows |
| **Legal Signal** | Phrases indicating importance (e.g., "we hold that", "ratio decidendi") — boost chunk scores |
| **Memo** | Structured research output generated by the research agent |
| **Multi-vector** | Multiple vector types per case: chunk, proposition, ratio, headnote, statute, summary, community |
| **Passage Expansion** | Using `char_start`/`char_end` metadata to expand a matched chunk to its surrounding context |
| **Proposition** | An atomic legal statement extracted from case text, stored as its own vector |
| **Provider** | Concrete implementation of an interface (e.g., `GeminiLLM` implements `LLMProvider`) |
| **Protocol** | Python `typing.Protocol` class defining an interface for dependency injection |
| **RAG** | Retrieval-Augmented Generation — LLM generates answers grounded in retrieved documents |
| **Reranker** | ML model (Cohere) that re-scores search results for relevance after initial retrieval |
| **RRF** | Reciprocal Rank Fusion — merges multiple ranked lists using `1/(k+rank)` scoring |
| **Send()** | LangGraph primitive for fanning out to multiple parallel worker nodes |
| **SSE** | Server-Sent Events — streaming protocol for real-time agent/chat updates |
| **StateGraph** | LangGraph's core abstraction — a directed graph of processing nodes with shared state |
| **Task Type** | Gemini embedding parameter: `RETRIEVAL_DOCUMENT` (indexing) vs `RETRIEVAL_QUERY` (searching) |
| **Tenacity** | Python retry library — exponential backoff for flaky external service calls |
| **tsvector** | PostgreSQL text search vector type — enables fast keyword search with stemming |
| **Vector Type** | Category of stored vector: `chunk`, `proposition`, `ratio`, `headnote`, `statute`, `summary`, `community` |
| **Worker** | Specialized search node in the research agent (e.g., case_law_worker, statute_worker) |

## Internal Naming Conventions

| Name | Refers To |
|------|-----------|
| `case_id` | UUID primary key in the `cases` table |
| `source_id` | S3 filename or external identifier for a case |
| `text_hash` | SHA-256 hash of case full text (deduplication) |
| `vector_type` | Pinecone metadata field distinguishing vector categories |
| `char_start` / `char_end` | Character offsets in original case text for passage expansion |
| `ingestion_status` | Case processing state: `pending`, `processing`, `completed`, `failed` |
| `search_degraded` | Flag indicating partial service failure during search |
| `treatment_warning` | UI warning that a cited case may be overruled |
| `bench_type` | Encoded bench size: `single`, `division`, `full`, `constitutional` |
| `coram_size` | Integer count of judges (1, 2, 3, 5, 7, 9, 11) |
| `is_reportable` | Boolean: whether the judgment was marked reportable by the court |
| `outcome_summary` | Brief description of the case outcome (allowed/dismissed/etc.) |
