# AUDIT MAP — Smriti Codebase Function Inventory

> Generated by AUDIT-1 through AUDIT-10. Each function is mapped to its legal workflow purpose.

---

## backend/app/core/search/

### hybrid.py
- `rrf_merge()` — Reciprocal Rank Fusion merge of multiple ranked result lists with optional per-list weights. **Legal workflow: reranking**
- `hybrid_search()` — Execute parallel vector+FTS search, RRF merge, Cohere rerank, and PostgreSQL enrichment with Redis cache and outcome bias detection. **Legal workflow: case search**
- `invalidate_search_cache()` — Scan and delete all search/suggest cache keys from Redis using pattern matching. **Legal workflow: caching**

### query.py
- `expand_statute_references()` — Expand old-law/new-law statute references (IPC↔BNS, CrPC↔BNSS, IEA↔BSA) bidirectionally for search. **Legal workflow: statute lookup**
- `understand_query()` — Parse raw search query into structured components (intent, entities, filters) via Gemini structured JSON, with LLM-failure passthrough. **Legal workflow: case search**

### fulltext.py
- `search_fulltext()` — Execute PostgreSQL FTS on searchable_text tsvector with ts_rank_cd ranking, supports judgment section filtering and quoted phrase matching. **Legal workflow: case search**

### semantic_cache.py
- `SemanticCache.get()` — Check Redis Stack HNSW index for semantically similar queries (cosine > 0.92 threshold), return cached memo if matched. **Legal workflow: caching**
- `SemanticCache.put()` — Store query embedding and metadata in Redis HNSW index for future semantic matching. **Legal workflow: caching**

## backend/app/core/ingestion/

### pdf.py
- `clean_extracted_text()` — Normalizes extracted PDF text with NFKC Unicode normalization, zero-width removal, header deduplication, and whitespace cleanup. **Legal workflow: judgment parsing**
- `reattach_footnotes()` — Detects footnote definitions in judgment text and reattaches them inline near references. **Legal workflow: judgment parsing**
- `extract_pdf_text()` — Async wrapper for text extraction with OCR fallback, runs blocking I/O in thread pool. **Legal workflow: judgment parsing**
- `extract_with_ocr()` — Full-document fallback OCR extraction for scanned PDFs using pdf2image and pytesseract. **Legal workflow: judgment parsing**
- `score_text_quality()` — Tiers extracted text as high/medium/low based on character count, legal keyword density, and alpha ratio. **Legal workflow: judgment parsing**
- `assess_extraction_quality()` — Checks alphabetic character ratio and legal term presence to flag OCR garbage. **Legal workflow: judgment parsing**
- `extract_and_score()` — Orchestrates text extraction, OCR fallback, and quality scoring into single TextQuality object. **Legal workflow: judgment parsing**
- `extract_tables()` — Extracts tabular data from PDFs using pdfplumber's table detection. **Legal workflow: judgment parsing**

### chunker.py
- `detect_judgment_sections()` — Detects structural sections (FACTS, ANALYSIS, ORDER, DISSENT, etc.) via regex pattern matching at heading positions. **Legal workflow: judgment parsing**
- `chunk_judgment()` — Chunks judgment text respecting section boundaries with 2000-char size, 200-char overlap, sentence-boundary awareness. **Legal workflow: judgment parsing**

### metadata.py
- `extract_metadata_llm()` — LLM-based structured metadata extraction with Gemini PDF multimodal fallback to text-only. **Legal workflow: metadata extraction**
- `compute_extraction_confidence()` — Computes 0.0-1.0 confidence score weighting critical fields (title, citation, court, year). **Legal workflow: metadata extraction**
- `validate_with_regex()` — Validates LLM-extracted metadata with deterministic checks (year bounds, date format, enum values, list types). **Legal workflow: metadata extraction**
- `validate_cross_fields()` — Cross-validates fields against each other (year vs decision_date, bench_type vs judge_count, etc.). **Legal workflow: metadata extraction**
- `normalize_case_type()` — Maps case type abbreviations and variants to 20+ canonical forms. **Legal workflow: metadata extraction**
- `validate_parquet_data()` — Cleans Parquet metadata before merge (NaN → None, invalid dates, year ranges). **Legal workflow: metadata extraction**
- `merge_metadata()` — Merges Parquet ground truth with LLM extraction (parquet wins for title/citation/court, LLM wins for ratio/acts/keywords). **Legal workflow: metadata extraction**

### contextual_embeddings.py
- `generate_contextual_prefix()` — Generates LLM-prefixed contextual sentence for a single chunk (case law or statute). **Legal workflow: embedding generation**
- `batch_contextualize_chunks()` — Parallel batch contextualisation of chunks with rate limiting and failure handling. **Legal workflow: embedding generation**

### anonymizer.py
- `anonymize_text()` — Masks Aadhaar/PAN/phone numbers using regex patterns; returns (masked_text, was_modified). **Legal workflow: judgment parsing**
- `detect_sensitive_case()` — Detects POCSO/sexual assault cases by checking acts_cited for sensitive statutes. **Legal workflow: judgment parsing**

### section_summarizer.py
- `generate_section_summaries()` — Generates Level-1 section summaries (2-4 sentences) per judgment section type using Flash LLM. **Legal workflow: embedding generation**
- `build_pinecone_summary_vectors()` — Constructs Pinecone upsert records for Level-1 summary vectors with enriched metadata. **Legal workflow: embedding generation**

### rate_limiter.py
- `AsyncRateLimiter` — Token-bucket style async rate limiter for API calls (default 30 RPM). **Legal workflow: infrastructure**
- `RateLimiterPool` — Per-API-key rate limiter pool for key rotation support. **Legal workflow: infrastructure**

### graph_retry.py
- `record_graph_failure()` — Records failed citation graph build to retry queue with error message. **Legal workflow: graph operations**
- `get_pending_retries()` — Returns list of (case_id, retry_count) pairs pending graph rebuild. **Legal workflow: graph operations**
- `mark_retry_success()` — Removes case from retry queue after successful graph build. **Legal workflow: graph operations**
- `increment_retry_count()` — Increments retry counter for failed graph build attempt. **Legal workflow: graph operations**

### pipeline.py
- `ingest_judgment()` — Full 9-step pipeline: PDF extract → metadata merge → validate → store → PostgreSQL → chunk → embed → Pinecone → Neo4j. **Legal workflow: ingestion orchestration**
- `bulk_upsert_cases()` — Bulk upserts pre-processed cases to PostgreSQL with transaction batching. **Legal workflow: ingestion orchestration**
- `ingest_batch()` — Orchestrates multi-case batch ingestion with progress tracking and error handling. **Legal workflow: ingestion orchestration**
